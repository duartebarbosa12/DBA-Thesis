# DBA-Thesis
The full repository for my DBA Thesis: "Volatility Risk Premium: A Machine Learning Approach to Option Straddle Strategies on US Equities"

# Abstract
Option straddles allow investors to trade volatility rather than directional price movements. The gap between implied volatility, which reflects the market’s expectation of future volatility, and realized volatility is commonly referred to as the volatility risk premium. Capturing this premium requires not only accurate volatility forecasts but also a trading framework that can translate forecasts into executable and profitable positions. In quantitative finance, volatility forecasting and options strategy evaluation are often studied separately, which makes it difficult to assess whether predictive accuracy leads to implementable trading performance in real markets. A machine-learning approach to this problem is economically relevant because it can help traders diversify away from pure price prediction and toward systematic volatility-based strategies.

This thesis develops **Tradelity**, a three-layer system that forecasts volatility, generates trading decisions, and simulates a portfolio with actual CBOE option quotes. The system is trained based on 6 years of daily data (2018–2024) for 10 US equities and back tested in 7,308 options from December 2023, across 3 holding horizons. Over 50 models are compared using out-of-sample RMSE, and the best model’s forecasts are fed into a trading algorithm that constructs option straddles, opens long and short positions, and computes returns.

Regularization models showed the most potential. Specifically, Lasso Regression outperformed all other models. The 5-day holding horizon produced a 7.93% monthly return with a 52% win rate after transaction costs. While the 3-day horizon was marginally profitable, the 10-day horizon lost 9.06%, driven by concentrated short straddle losses.

These results demonstrate that machine-learning volatility forecasts can generate economically meaningful straddle trading decisions on US equities, provided the holding period is long enough for price movements to exceed premiums and transaction costs, and that short straddle exposure is carefully controlled.

# File Structure
- **DBA_Thesis_v9.ipynb:** The latest version of the notebook with the full implementation (except UI) of the methodology described in the thesis. This can be run from start to bottom provided all dependencies are installed
- **algorithm.py**, **app.py**, **computing_results.py:** The UI code, which can be run by app.py
- **Results Folder:** Contains all results generated by the notebook, included all tables and mid-diagnostic for deeper inspection
- **Data Folder:** Includes all *CBOE* data required to be loaded into looking environment inside a folder called "Data"
- **Templates Folder:** Covers all of the html and formatting files required for the dashboard
